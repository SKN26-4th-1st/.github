## Team Identity

## Core Capabilities

- 핵심 역량 정리는 현재 **미정**입니다.
- 프로젝트 요구사항 확정 후 업데이트할 예정입니다.
- Git 기반 협업 (PR / Issue / Conventional Commits)

---

## Working Method

- Issue 기반 작업 분리
- Branch 전략 준수
- Conventional Commit 사용
- PR 리뷰 후 Merge
- 프롬프트·RAG 설정·평가 결과 및 설계 근거 문서화

---

## Repository Philosophy

각 프로젝트는 독립된 저장소로 관리하며, 다음 원칙을 따릅니다:

- 구조화된 폴더 구성 (세부 구조는 프로젝트별로 확정)
- 데이터·문서·비밀값(`.env`) 분리 관리
- 설정·의존성·평가 항목 버전 관리
- 산출물 및 실험 기록 유지
